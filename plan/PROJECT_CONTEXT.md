# Контекст Проекта и Организация Памяти

Этот проект использует Obsidian в качестве интерфейса для управления задачами, а также агента (LLM) для автоматизации кодинга и работы с задачами.

## Структура директории `plan/`

* `inbox.md` - Входящие идеи, мысли, планы. Сюда пользователь пишет все неструктурированные задачи.
* `overview.md` - Назначение проекта и его глобальная цель.
* `history.md` - История развития проекта. Файл только для дописывания!
* `PROJECT_CONTEXT.md` - Этот файл (описание архитектуры памяти).
* `kanban_board.md` - Доска задач. Файл работает со стандартным плагином Kanban в Obsidian (содержит списки под заголовками `## backlog`, `## active`, `## done`).
* `tasks/` - Папка для задач. Здесь лежат задачи из inbox, оформленные в виде простых markdown-файлов.
* `docs/` - Свободная папка для документации (совещания, статьи, презентации и внешние ссылки).
* `prompts/` - Стандартные промпты для работы с памятью (например, "рассортировать inbox").
* `drafts/` - Сюда переносятся различные черновики и результаты рабочих промптов, которые лучше никому не показывать.

## Правила работы для LLM (Агента)

1. **Не смешивать контекст:** Во время выполнения конкретной задачи по коду, не редактируй файлы памяти (статусы, задачи, историю). Редактирование памяти происходит *отдельным запросом* (человек попросит "обнови статусы задач" или использует промпт из папки `prompts/`).
2. **Формат задач (Tasks):** Каждая задача в папке `tasks/` — это простой markdown-файл:
   ```markdown
   # Название задачи (информативное)
   Краткое описание эпика. Решения, ссылки и черновики пишутся в теле.
   ```
3. **Управление Kanban-доской:** Cтатус задачи определяtтся её местоположением на доске `plan/kanban_board.md`.
   - `backlog` — задача запланирована.
   - `active` — задача в работе.
   - `done` — задача завершена.
   При создании новой задачи в `plan/tasks/`, ты ОБЯЗАТЕЛЬНО должен добавить ссылку на нее в `plan/kanban_board.md` в соответствующую колонку в формате: `- [ ] [[название_файла]] Название задачи`.
   Для смены статуса нужно просто перенести эту строчку под другой заголовок (`## active`, `## done`) в файле доски.
4. **Крупноузловые задачи:** В базу задач (`tasks/`) заносятся только крупные задачи без деталей реализации. Мелкие шаги обсуждаются в контексте текущего диалога с LLM.
