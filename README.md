# Система ведения задач и идей

Простой шаблон для управления идеями, задачами и документацией проекта. Вся структура хранится в чистом Markdown в папке `plan/`. Файл `plan/kanban_board.md` может быть открыт в Obsidian (с плагином Kanban) для визуального удобства управления доской.

## Как начать (Быстрый старт)

**Вариант А: Для совершенно нового проекта**
1. Склонируйте этот репозиторий.
2. Удалите папку `.git` (`rm -rf .git`).
3. Инициализируйте новый чистый репозиторий (`git init`).
4. Начинайте работу!

**Вариант Б: Для переноса существующего проекта**
1. Скопируйте папку `plan/` в корень вашего старого проекта.
2. Используйте **Промпт 3 (Migrate Project)**, чтобы AI-агент проанализировал старый проект и заполнил документацию.
3. Используйте **Промпт 4 (Infinite Reflection)** для проверки качества переноса фактов.
4. Добавьте в свой `CLAUDE.md` или его аналог следующий текст:

```
Основной контекст и правила работы с документацией проекта описаны в файле:
`plan/PROJECT_CONTEXT.md`
В первую очередь всегда читай этот файл перед началом работы, чтобы понимать, как организована структура проекта.
```

Для визуализации доски вы можете открыть папку `plan/` как Vault в Obsidian и установить плагин Kanban (mgmeyers).

## Как работать с AI-агентом (Промпты)

В папке `plan/prompts/` находятся подготовленные сценарии (промпты) для отправки LLM (Claude Code / OpenClaude).

**1. Сортировка идей (Sort Inbox)**
*Промпт: `plan/prompts/1_Sort_Inbox.md`*
Если вы записали мысли в `plan/inbox.md`, отправьте этот промпт агенту. Он распарсит текст и **распределит** идеи: либо дополнит ими уже существующие markdown-файлы задач в `plan/tasks/`, либо создаст новые задачи и добавит их в Backlog на `plan/kanban_board.md`.

**2. Обновление статусов (Update Status)**
*Промпт: `plan/prompts/2_Update_Status.md`*
Запускается **в том же чате** сразу после того, как агент выполнил работу. Агент проанализирует изменения, допишет детали реализации внутрь файлов задач в `plan/tasks/` и обновит `plan/kanban_board.md`, переместив карточки в колонку `## active` или `## done`.

**3. Интеграция старого проекта (Migrate Project)**
*Промпт: `plan/prompts/3_Migrate_Project.md`*
Агент проанализирует указанный целевой проект, изучит заметки автора, историю Git и самостоятельно создаст структуру в `overview.md` и хронологию в `history.md`.

**4. Верификация (Reflect & Refine)**
*Промпт: `plan/prompts/4_Infinite_Reflection.md`*
Проверка работы после применения Промпта 3. Заставляет агента прочитать правила, перепроверить факты, сверить `history.md` с реальными логами Git и исправить выдуманные данные или упущения.

## Файлы и Папки
- `plan/PROJECT_CONTEXT.md` - Правила для AI-агента (важно для понимания структуры).
- `plan/overview.md` - Описание архитектуры и директорий проекта.
- `plan/history.md` - История разработки (только добавление информации).
- `plan/kanban_board.md` - Kanban-доска (Markdown, работает с плагином в Obsidian).
- `plan/inbox.md` - Входящие идеи. **Сюда нужно писать новые неструктурированные мысли и идеи**, которые агент позже рассортирует.
- `plan/tasks/` - Папка с задачами (каждая задача — отдельный markdown файл).
- `plan/docs/` - Свободная папка для документации (записи совещаний `meetings`, статьи `papers`, презентации и т.д.).
- `plan/drafts/` - Папка для черновиков и результатов работы промптов, которые вы не хотите показывать публично. **Внимание:** содержимое этой папки находится под `.gitignore` (кроме файла `example.md`), поэтому ваши черновики не попадут в историю коммитов.
- `plan/prompts/` - Папка для промптов. Вы можете добавлять сюда **свои собственные промпты** для автоматизации взаимодействия с системой.
